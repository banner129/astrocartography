## 四、SEO 技术规范

### 4.1 元数据优化

#### Title 标签规范
- 长度: 50-60 字符
- 格式: `[目标关键词] - [价值主张] [年份]`
- 示例: "Free Astrocartography Calculator - Generate Your Chart Online 2025"

#### Meta Description 规范
- 长度: 150-160 字符
- 包含: 目标关键词 + 吸引点击的描述
- 示例: "Use our free astrocartography calculator to generate your personalized chart. No signup required. Get instant results with professional-grade calculations."

#### Canonical URL
- 每个页面必须设置 Canonical URL
- 格式: `https://astrocarto.org/[locale]/[path]`
- 避免重复内容问题

### 4.2 HTML 语义化

#### 标题层级
- H1: 页面唯一，包含完整目标关键词
- H2: 3-5 个，自然重复关键词
- H3: 支持 H2 内容，使用关键词变体

#### 语义化标签
- `<article>`: 博客文章
- `<section>`: 内容区块
- `<nav>`: 导航菜单
- `<aside>`: 侧边栏内容

### 4.3 关键词优化

#### 关键词密度
- 目标关键词: 1-2%
- 次要关键词: 0.5-1%
- 避免关键词堆砌

#### 关键词位置
- 前 100 字必须出现目标关键词
- URL 中包含关键词
- 图片 alt 属性包含关键词
- 内链锚文本使用关键词

### 4.4 内链策略

#### 内链数量
- 每个页面至少 3-5 个内部链接
- 使用描述性锚文本（包含关键词）
- 建立主题集群（Topic Cluster）

#### 内链结构示例
首页 (astrocartography)
↓
工具中心 (astrocartography calculator)
├─→ 学习中心 (what is astrocartography)
│ ├─→ 教程 (how to read map)
│ └─→ 行星线专题 (venus line)
│
└─→ 行星线专题 (sun line)
├─→ 工具页 (calculator)
└─→ 学习中心 (meanings)


### 4.5 结构化数据

#### 页面类型对应的 Schema
- **工具页:** WebApplication schema
- **博客页:** Article schema
- **教程页:** HowTo schema + Article schema
- **FAQ 页:** FAQPage schema
- **总览页:** WebPage schema

#### 面包屑导航
- 所有页面添加 BreadcrumbList schema
- 格式: Home > Category > Page

### 4.6 性能优化（Core Web Vitals）

#### 指标要求
- LCP (Largest Contentful Paint): < 2.5s
- FID (First Input Delay): < 100ms
- CLS (Cumulative Layout Shift): < 0.1

#### 优化措施
- 图片优化和懒加载
- 代码分割和按需加载
- 使用 Next.js ISR 缓存
