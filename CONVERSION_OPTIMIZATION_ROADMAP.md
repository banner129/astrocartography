## 二、执行优先级与排期

| 优先级   | 功能模块 | 预期影响 | 工作量 |
|----------|----------|----------|--------|
| **P0 紧急** | 修复支付按钮偶发性失效 | 直接减少付费漏损 | 小（排查 Creem/支付日志、FedCM、前端 handleCheckout） |
| **P0 紧急** | 修复星座计算错误 | 避免差评影响信任 | 小（排查 calculate-astrocartography、时区/宫位） |
| **P1 高** | 后端实现「第 1 次免费问答」不登录可收到回复 | 用户先有 1 次完整体验再遇付费墙 | 小（仅改 /api/astro-chat 一处判断） |
| **P1 高** | 付费触发文案个性化：从「积分不足」改为情感引导 | 降低付费墙阻力 | 小（<3 天） |
| **P1 高** | 改为两段式免费体验 + 情绪高点再弹付费引导 | 预计转化率提升 50–100% | 中（1–2 周） |
| **P1 高** | （可选）AI 自动生成星图摘要（不耗积分） | 提升首屏价值感知 | 中（1–2 周） |
| **P2 中** | 新增 $4.9 尝鲜包，调整定价页布局 | 降低首次付费门槛 | 小（<1 周） |
| **P2 中** | AI 回答内置三段式悬念结构，优化 System Prompt | 提升对话深度和积分消耗 | 中（1 周） |
| **P2 中** | 图表图例：每条线/颜色含义说明（常驻 + 线 Popup） | 解决「看不懂图」导致的流失 | 小（<1 周） |
| **P3 低** | 图表新手引导（高亮 3 条线 + 浮层 + 「问 AI」） | 降低新用户流失 | 中（1–2 周） |
| **P1 高** | 聊天内常显积分余额（登录用户在 Chat 顶部/输入框旁看到「剩余 X 积分」） | 用户清楚剩余次数，减少突然被拦感，促主动购买 | 小 |
| **P1 高** | 付费墙关闭后的软提醒：再次打开 Chat 时轻量提示「0 积分，解锁 $4.9」+ 按钮 | 二次触达未付费用户 | 小 |
| **P2 中** | 支付成功页：明确「已到账 X 积分」+ CTA「返回星图继续提问」 | 付完立刻再用一次，强化价值感 | 小 |
| **P2 中** | 定价/弹窗加社会证明（如「24,000+ 用户已生成星图」） | 降低决策犹豫 | 小 |
| **P2 中** | 用尽免费未付费的挽回：邮件或再访时软提示（cookie 判断 + 轻量文案） | 挽回放弃用户 | 中 |
| **P2 中** | 定价页信任文案（退款、免责、隐私），限制说明移至 FAQ/图表页 | 减少焦虑 | 小 |
| **P2 中** | 支付步骤精简（少一步是一步，可选 Apple/Google Pay） | 减少支付流失 | 中 |
| **P3 低** | 漏斗埋点（进入 chart→点 Ask AI→发问→见付费墙→点购买→完成支付） | 找准流失环节，数据驱动优化 | 小 |
| **P3 低** | 小流量 A/B 测试（1 次 vs 2 次免费、$4.9 有无、不同引导文案） | 用数据选长期方案 | 中 |

## 三、分步实施顺序（建议）

按对系统冲击从小到大的顺序执行，每步可单独上线验证：

```
阶段 0：止血
  0.1 排查「点购买没反应」→ 0.2 修支付漏损 → 0.3 星座计算核查

阶段 1：免费 1 次真正生效
  1.1 后端允许第 1 次不登录放行 → 1.2 前端确认传参

阶段 2：付费墙文案软化
  2.1 积分不足弹窗改为个性化/情感引导文案 → 2.2 多语言 key

阶段 3：免费 2 次 + 情绪高点弹付费
  3.1 后端支持 2 次免费 → 3.2 前端 FREE_QUESTIONS_LIMIT=2 → 3.3 第 2 次结束后先弹引导再登录

阶段 4：图表可读
  4.1 图例（行星/线含义）→ 4.2 线 Popup 加一句含义 → 4.3（可选）首次高亮 + 「问 AI」

阶段 5：定价与低价入口
  5.1 定价页文案调整 → 5.2 新增 $4.9 档位 → 5.3 定价出现时机（用完免费后再突出）

阶段 6：AI 回答结构
  6.1 System Prompt 三段式（满足感 + 悬念 + 引导）
```
