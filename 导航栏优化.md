# 导航栏优化.md

## 背景

当前网站已经有一定自然搜索流量，尤其是 `/astrocartography-lines/` 页面已经开始获得 Google 展示和点击。根据 Search Console 导出的数据，这个页面过去 3 个月大约有：

- 点击：54
- 展示：7055
- 点击率：0.77%
- 平均排名：8.78

因此，导航和页面结构优化必须以“稳妥增强”为原则，避免一次性大改导致 Google 重新理解页面主题、站内权重和用户路径。

## 核心原则

1. 不改已有 URL。
2. 不改顶部一级导航名称，尤其是 `Lines`。
3. 不删除已有 line 页面。
4. 不大幅修改 `/astrocartography-lines/` 的标题方向和核心正文。
5. 不把已有正文替换成纯目录页。
6. 先增强 hub 页，再考虑压缩导航菜单。
7. 每次只做小范围变更，观察 2-4 周数据后再继续。

## 当前推荐策略

最稳妥的方式是：先把 `/astrocartography-lines/` 改造成一个更清晰的聚合页，也就是 Astrocartography Lines Hub。

顶部导航暂时保持现状：

- `Home`
- `Tools`
- `Chart`
- `Lines`
- `Blog`

`Lines` 下拉菜单可以暂时继续列出全部入口，包括：

- All Astrocartography Lines
- Where to Live
- Sun Line
- Moon Line
- Venus Line
- Mars Line
- Jupiter Line
- Saturn Line
- Mercury Line
- Uranus Line

这样做的好处是，短期内不改变用户和 Google 已经熟悉的主导航入口。

## 为什么先改 Hub 页

从 Google SEO 角度，站内链接结构会影响 Google 对页面重要性的理解。更清晰的结构应该是：

```text
Home
  -> Astrocartography Calculator
  -> Astrocartography Lines Hub
       -> Planetary Line Pages
       -> Life Goal Pages
       -> Angle Pages
       -> Location Decision Pages
  -> Chart Calculators
  -> Blog / Guides
```

`/astrocartography-lines/` 应该承担“分类聚合页”的作用，而不是只作为一篇长文章存在。

类似 Canva 的模板聚合页逻辑是：

- 顶部说明这个集合是什么
- 提供清晰的分类入口
- 用卡片或分组帮助用户快速下钻
- 页面下方保留解释型 SEO 内容
- 用相关探索入口承接更多长尾需求

这个思路适合迁移到 AstroCarto，但视觉不需要照搬 Canva。

## 分阶段执行计划

### 第一阶段：不动导航，只增强 Hub 页

目标：让 `/astrocartography-lines/` 具备真正的聚合页能力。

执行项：

- 保留 H1、title、description 的核心方向
- 在首屏下方新增 `Start Here`
- 新增 `Browse by Planetary Line`
- 新增 `Browse by Life Goal`
- 新增 `Browse by Angle`
- 原正文和 FAQ 保留
- 新增 `CollectionPage` 和 `ItemList` schema

这一阶段是最稳妥的，不会改变已有 URL，也不会减少已有内容。

### 第二阶段：观察 2-4 周

重点观察 Search Console：

- `/astrocartography-lines/` 展示是否上升
- CTR 是否改善
- 平均排名是否稳定
- line 子页面点击是否增加
- `/astrocartography-where-to-live/` 是否开始获得展示
- `mc line`、`ic line`、`asc line` 相关查询是否变多

如果数据稳定或上升，再进入第三阶段。

### 第三阶段：再考虑压缩顶部导航

当 hub 页已经能很好承接所有 line 页面后，再考虑把顶部 `Lines` 下拉从“全部列出”调整为“核心入口”。

可能的稳妥版菜单：

- All Astrocartography Lines
- Where to Live
- Venus Line
- Jupiter Line
- Moon Line
- Saturn Line

不要一开始就删很多顶部入口。应该先让 hub 页承担导航，再逐步压缩顶部菜单。

这些页面应该从 `/astrocartography-lines/` hub 页自然分发权重。

## 不建议做的事情

短期内不建议：

- 改 `/astrocartography-lines/` URL
- 大幅改 H1 和 title
- 删除原有正文
- 把页面改成纯卡片目录
- 一次性移除大量顶部菜单入口
- 新建太多空泛页面
- 做没有实际内容承接的分类入口

## 推荐结论

当前最稳的路线是：

```text
先不动顶部导航
先增强 /astrocartography-lines/ 为 hub page
保留已有 SEO 内容
新增分类导航和 ItemList
观察 2-4 周数据
再决定是否压缩顶部 Lines 下拉菜单
```

这个改法既符合 Google 对清晰站内结构的偏好，也能降低已有流量波动风险。
