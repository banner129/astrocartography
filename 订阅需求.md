# 一、功能概述

## 这件事是什么？

在 **不取消、不改动** 现有 Standard（$9.9）和 Professional（$18.9）一次性套餐的前提下，**新增按月/按年的订阅套餐（Plus）**，让用户除了「一次性买断 AI 次数」，还可以选择「每月付一点、持续回来研究搬迁」。

## 为什么要做？

| 现在的情况 | 带来的问题 |
|-----------|-----------|
| 地图、城市对比免费 | 用户用完就走，粘性低 |
| 只卖一次性 credits | 用完没有理由再回来 |
| Chart 不能保存到账户 | 用户反复抱怨（feedback 里很常见） |
| 没有订阅 | 没有每月固定收入 |

## 已经定好的原则

1. **一次性套餐照旧**：价格、权益都不动  
2. **免费功能照旧**：地图、城市对比继续免费  
3. **永久对话历史只给 Professional 一次性 $18.9**：Plus 订阅 **不给** 这个，避免和 $18.9 抢卖点  
4. **订阅过期 → Profile 直接冻结**；续订 → 立刻解冻；数据不删  
5. **没有白名单、没有内测**：做好就全量上线；靠「先上小的、观察几天、再加大功能」来控制风险  

## 三句话定位

| 类型 | 卖给谁 | 一句话 |
|------|--------|--------|
| 一次性 Standard | 这次决策够用就行 | $9.9 买断约 100 次 AI |
| 一次性 Professional | 重度决策 + 要永远翻历史 | $18.9 买断约 300 次 AI + **永久对话历史** |
| 订阅 Plus | 还在选城市、会持续研究 | $6.9/月，每月 50 次 AI |

> **阶段一文案说明：**「还在选城市」= 试探型月付（低门槛先试）；「持续研究」= 阶段二 Profile 上线后的叙事，阶段一定价页不写。

# 二、整体实现的目标

## 2.1 最终要建成什么样？

网站有三条并行的付费路径，用户按自己的场景选，互不替代：

```
免费用户  →  一次性买断（Standard / Professional）  →  订阅（Plus / 以后的 Pro）
  拉新              这次决策型用户                        持续研究型用户
```

## 2.2 最完整的套餐（全部做完之后的样子）

> 注意：这不是第一版就要全上的。实际按「三、阶段任务」一步一步来。

### 免费版

- 无限生成地图  
- 城市对比（免费）  
- 注册送有限 AI 次数  
- 不能保存 Profile  
- 不能看对话历史  
- 没有 PDF  

### 一次性 Standard — $9.9（已有，不改）

- 约 100 次 AI（买断，用完为止）  
- PNG 地图下载、TXT 导出  

### 一次性 Professional — $18.9（已有，不改）

- Standard 的全部  
- 约 300 次 AI（买断）  
- **永久对话历史**（这是它独有的卖点，订阅不给）  

### 订阅 Plus — $6.9/月 或 $59/年

| 权益 | 什么时候上 |
|------|-----------|
| 每月 50 次 AI | **阶段一** |
| PNG / TXT 导出 | **阶段一** |
| Profile 保存（最多 3 个） | **阶段二** |
| 城市收藏（最多 5 个） | **阶段二**（可选） |
| 对话历史 | ❌ 永远不给（避免和 $18.9 重复） |

### 订阅 Pro — $12.9/月 或 $99/年（更后面）

| 权益 | 什么时候上 |
|------|-----------|
| 每月 150 次 AI | **阶段三** |
| Profile 最多 10 个 | **阶段三** |
| PDF 报告（爱情/事业/家园） | **阶段三** |
| AI 按目标推荐城市（Love/Job/Home） | **阶段三** |
| 月度 Transit 简报 | **阶段三** |
| 订阅期内对话历史 + 导出 | **阶段三**（注意：仍不等于一次性的「永久」） |

### 阶段四（长期）

- 双人/家庭 Profile 对比  
- AI 记住用户偏好  
- Relocation 深度图（Paran 等）  
- 从业者帮客户算（多 Profile、白标 PDF）  

## 2.3 定价页怎么摆？

分两块，不要五个套餐排一排：

```
┌──────────────────────────────────────────┐
│  一次性 · 这次决策                         │
│  Standard $9.9    Professional $18.9      │
├──────────────────────────────────────────┤
│  订阅 · 持续研究                           │
│  Plus $6.9/月   （Pro 先写「即将推出」）    │
└──────────────────────────────────────────┘
```

## 2.4 业务上想达成什么？

| 目标 | 怎么判断做到了 |
|------|---------------|
| 有人愿意订阅 | 上线后真实订阅 ≥ 2～3 单 |
| 一次性没被抢走 | Standard + Pro 周订单降幅 < 15% |
| 用户愿意回来（阶段二起） | 订阅用户存了 Profile，7 天内再回来 |
| 长期有稳定收入 | 订阅收入（MRR）逐步增加 |

# 三、阶段任务

> **你的上线方式：** 每个阶段开发完 → 直接全量对所有用户开放 → 观察几天 → 达标了再做下一阶段。没有白名单，没有内测账号。

## 阶段零：开发前准备

**任务名称：** 后台配置（用户看不到任何变化）

**要做什么：**

- 在 Creem 后台创建 Plus 月付、年付两个订阅产品，拿到 product ID  
- 确认命名：`plus-monthly`、`plus-yearly`  
- **不发邮件**（2026-07 刚发过调研）  

**改什么：** 只改 Creem 后台和配置文件，网站对用户无变化  

**观察什么：** 无（还没上线）

---

## 阶段一：先上订阅套餐

**任务名称：** Plus 订阅上线（最小版，只验证「有人愿意月付」）

**这个阶段用户能买到什么：**

- Plus：$6.9/月 或 $59/年  
- 每月 50 次 AI  
- PNG / TXT 导出  
- **没有** Profile 保存  
- **没有** 对话历史  
- 地图页 **没有任何改动**  

**改什么：**

| 改哪里 | 做什么 |
|--------|--------|
| 定价页 | 加 Plus 订阅卡片，和一次性分两块展示 |
| 支付 Checkout | 支持 Plus 月付、年付 |
| Webhook | 处理订阅开通、续费、取消、过期 |
| 权益系统 | 订阅有效 → 每月发 credits；可用 PNG/TXT |
| 用户中心（简单版） | 显示：是否在订、下次续费日、本月剩多少 AI |

**不改什么：**

- 地图页（不加 Save 按钮）  
- Standard / Professional 一次性（价格、逻辑都不动）  
- 免费地图、城市对比  
- 不发邮件（静默上线）  

**上线方式：** 开发完 → **直接全量开放**

**观察什么（至少 7 天）：**

| 观察项 | 达标 | 未达标怎么办 |
|--------|------|-------------|
| 有没有人买订阅 | ≥ 2～3 单真实订单 | 0 单 → 改定价页文案，**不做阶段二** |
| 一次性有没有掉 | Standard+Pro 周订单比上线前降 < 15% | 降太多 → 改说明、延长观察 |
| 支付是否正常 | credits 到账、无投诉 | 有问题 → 先修，观察从修好那天重算 |

**进入阶段二的条件（三条同时满足）：**

1. 订阅 ≥ 2～3 单  
2. 一次性订单没有明显下滑  
3. 支付链路稳定  

### 阶段一文案策略（定价页）

**目标：** 验证有人愿意月付（≥2～3 单），不抢一次性（Standard+Pro 周订单降幅 <15%）

**一次性（Standard / Professional）：** 主体不动，仅轻量加固价值感  
- Standard tip 强调「100 次永久有效，比连续订 2 个月 Plus 更划算」  
- Professional 突出「永久对话历史」为专属权益（订阅不包含）

**Plus 只宣传阶段一真实权益（写进 features）：**  
- 每月 50 次 AI  
- PNG / TXT 导出  
- 随时取消  

**Plus 阶段一不宣传（不写进 features）：**  
- 「持续研究」（阶段二 Profile 上线后再写）  
- 优先 AI、邮件支持（Professional 专属）  
- 与 Standard 比「更划算」（数学上打不过 $9.9）  
- 年付不承诺未来功能（如不写「即将支持 Profile」）

**Plus 核心叙事：**  
- 月付 = 低门槛试水（$6.9 先试，不必一次付 $9.9）  
- 年付 = 省约 29%（约合 $4.9/月），面向确定用满一年的用户  

**路线图提示（仅月付 tip 小字，模糊、不具名）：**  
- 中文：「更多订阅功能陆续上线」  
- 英文：*More subscriber features coming soon*  
- 不点名 Profile / PDF，不承诺时间；年付不加此句  

**0 单时的调整顺序：**  
1. 先改文案（加强「$6.9 先试」）  
2. 再考虑价格（如 $4.9/月试水价）  
3. 不达标不做阶段二  

**阶段二起文案变更：** Profile 上线后，将「保存 Chart」正式写入 Plus features（属已交付功能，非承诺）

## 阶段二：加 Profile 保存

**任务名称：** 订阅用户可保存 Chart（拉回访）

**前置条件：** 阶段一观察达标

**这个阶段新增什么：**

- 订阅用户最多保存 **3 个** Profile（出生信息）  
- 地图页加 **Save / Load** 按钮  
- 免费用户点 Save → 引导去订阅  
- 城市收藏最多 5 个（可选，和 Profile 一起做或略晚）  
- 用户中心可管理 Profile 列表  
- **订阅过期 → Profile 直接冻结**；续订 → 立刻解冻；数据不删  

**改什么：**

| 改哪里 | 做什么 |
|--------|--------|
| 数据库 | 新建 Profile 表 |
| API | Profile 增删改查 |
| 地图页 | Save / Load 按钮 |
| 用户中心 | Profile 管理页 |
| 权益系统 | 只有有效订阅能 Save；过期冻结 |
| 定价页 | 文案加上「保存你的 chart」 |
| 邮件 | 上线前 3～5 天发 1 封功能通知 |

**观察什么（约 4 周）：**

| 观察项 | 目标 |
|--------|------|
| 订阅用户有没有存 Profile | ≥ 60% 存了至少 1 个 |
| 存了的人会不会回来 | 7 天内再打开 ≥ 35% |
| 一次性收入 | 不能比阶段一再掉太多 |

## 阶段三：Pro 档 + 内容功能

**任务名称：** 上 Pro 订阅、PDF、AI 推荐、Transit

**前置条件：** 阶段二稳定，订阅有一定存量

**这个阶段新增什么：**

- Pro 订阅：$12.9/月 或 $99/年  
- PDF 报告（爱情 / 事业 / 家园）  
- AI 目标导航：选 Love/Job/Home → 推荐 Top 5 城市  
- 月度 Transit 简报（Pro 专属）  
- 对比结果可保存/导出  
- Pro 订阅期内可看对话历史 + 导出（仍 ≠ 一次性的「永久」）  

**改什么：** 定价页加 Pro、Webhook 加 Pro 产品、PDF 生成、AI 推荐、Transit 定时任务、权益扩展

**观察什么：**

| 观察项 | 目标 |
|--------|------|
| 订阅月留存 | ≥ 55% |
| 新订阅里 Pro 占比 | ≥ 25% |
| 订阅用户用过 PDF | ≥ 30% |
